# 0.3.0 Runtime Verification — Perception Roll Requests

Status: **PASSED — runtime-verified live (2026-06-16, socketlib, two-client); `v0.3.0` tagged.**
See the "Verification PASSED — 2026-06-16" section below. Built on `docs/0.3.0-PROBE.md` (confirmed
`pf1ActorRollSkill` payload).

## Setup
- Module enabled + "Enable Perception Reveals" ON.
- GM seat + at least one player seat (e.g. Pants) whose user has an **assigned character**.

## Happy path
- [ ] GM marks a token undetected; the player does not see it.
- [ ] GM clicks **"Request a Perception check"** on the undetected token → dialog shows a DC input
      + eligible players (checked by default). Set a DC (e.g. 15) and Send.
- [ ] The targeted player gets a **"Perception Check"** prompt → clicks roll → the PF1 roll dialog
      appears → completes. The roll posts to chat as normal.
- [ ] total ≥ DC → GM sees a success toast; the token becomes visible to that player; the player
      shows as spotted (`source: perception`) under Manage Spotted.
- [ ] total < DC → GM sees a fail toast; the token stays hidden for that player.

## DC / manual fallback
- [ ] Leaving the DC blank → on a relayed roll the GM sees "no hidden DC set — use Manage Spotted"
      (no auto-spot; manual adjudication still works).

## Privacy (critical — inspect from the PLAYER seat)
- [ ] The hidden DC never appears in any world/client setting, flag, chat message, or socket
      payload. (Request payload = `{sceneId, tokenId, userIds}`; result = `{ids, actorName, total}`.)
- [ ] `publicRegistry` holds only safe state — `spottedBy` entries with `source:"perception"`,
      never a `dc` field.

## Fail-closed
- [ ] No active GM online → a relayed result is not adjudicated (nothing gets spotted).
- [ ] `pf15DiscoveryVeil._state.isActiveGMClient()` is false on the player; a hidden DC read on a
      non-active-GM client returns null.

## Safety
- [ ] No console errors on GM or player during the flow.
- [ ] Player reload preserves spotted state and visibility.
- [ ] A player whose user has no assigned character is warned (not silently broken).

## Known limitations (by design / MVP)
- Single-GM assumption: the GM who sets the DC must be `game.users.activeGM` (the same client that
  brokers results). Multi-GM tables fall back to the manual "no DC found" path.
- No automatic global reveal on success (the GM still decides when the party reveals globally).
- The roll uses `perceptionActorForUser(user)` — the assigned character if player-owned, else the
  first player-owned actor the user owns (eligibility is by **ownership**, per the 2026-06-15 fix).

## Verification PASSED — 2026-06-16 (socketlib transport, live cross-client)

Root cause of the earlier failures: the manifest was missing the top-level `"socket": true`
flag, so Foundry never created the module's socket namespace — raw `game.socket` module messages
were silently undelivered, and socketlib refused to register ("Please set socket:true in your
manifest"). Fixed by adding `"socket": true` (commit 822aaff) and migrating the transport to
socketlib (commit 6212ded). Both require a Foundry **server restart** (not just a browser reload)
to take effect.

After the server restart, verified live with GM "Dade" + player "DXWarlock" (both clients):

- socketlib registered cleanly on both clients (`manifest socket:true`, `socketlib.registerModule`
  returns a socket, no errors).
- **GM → player:** `executeForUsers("perceptionRequest", ...)` delivered — the "Perception Check"
  prompt appeared on the player's screen.
- **player → GM:** `executeAsGM("perceptionResult", { total: 22 })` delivered — the GM broker
  compared total 22 >= DC 12 and marked the player spotted
  (`spottedBy[userId] = { actorName: "Pants", source: "perception" }`).
- **Privacy held:** the hidden DC never appeared in the public registry (`findForbiddenKeys` = []).
- Roll-capture (`pf1ActorRollSkill` -> `chatMessage.rolls[0].total`) was confirmed by the 0.3.0 probe.
- All test state was restored afterward.

Result: 0.3.0 Perception roll requests work end-to-end via socketlib. Tagged `v0.3.0`.
