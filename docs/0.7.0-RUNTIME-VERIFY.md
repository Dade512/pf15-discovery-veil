# 0.7.0 Runtime Verification — Stripped public spell mechanics (opt-in)

Status: **PASSED — live, active-GM client (2026-06-17).** Driven via in-page evaluation on the live
GM client (Vivaldi, world `baphomets-fall-dev`), Foundry `13.350` / PF1 `11.11`. The masking path
(`onPreActionUse` → `postGenericCard` → `strippedEffectLabel`) was exercised with real Oswald spell
actions (Fireball = save, Magic Missile = attack). All test cards + gates were cleaned up afterward
(chat returned to its 86 baseline, spell gates 0, settings restored).

## Feature
A new **opt-in** world setting `spellPublicEffect` ("Show Masked Spell Effect"), OFF by default. When
ON, the generic masked-spell card gains a non-identifying effect line so players know what to brace
for. It surfaces ONLY: the **save type** (Fortitude/Reflex/Will, from `action.save.type` localized via
`pf1.config.savingThrows`), whether the action is a **spell attack** (`actionType ∈
{rsak,msak,rwak,mwak}`), and whether it **deals damage** (`damage.parts.length`). It NEVER surfaces
the spell name, school, save DC, or damage type/amount.

## What was verified

### Effect line content (PASS)
- **Fireball** (save-based, Reflex half, damage), setting ON → card text:
  `A spell is being cast. Effect: Reflex save · deals damage`.
- **Magic Missile** (ranged spell attack, damage), setting ON → card text:
  `A spell is being cast. Effect: spell attack · deals damage`.
- **Fireball**, setting OFF → card text: `A spell is being cast.` (no effect line — default behavior
  is unchanged).

### Privacy (the point — PASS)
For every card above, with the setting ON, all leak checks were **false**:
- no spell **name** ("Fireball" / "Magic Missile") in the content,
- no **school** (evocation/…),
- no **save DC** (`/DC\s*\d/`),
- no **damage amount** (`/\d+d\d+/`),
- the card **flags** carry no identity (`{castId, kind:"maskedSpell"}` only — `flagsHasIdentity:false`).

The disclosed facts (save type, attack, damage) narrow but do not reveal the spell, which is why the
setting is off by default and its hint states the trade-off. The hidden identity (name/school/DC)
still lives only in the active-GM private store (the 0.5.0 invariant is untouched — the effect line
reads the live action, never the private store, and writes nothing new).

## Acceptance — met
- Opt-in, off by default: no behavior change for existing tables. ✔
- When on, players see a non-identifying save/attack/damage summary on the generic card. ✔
- Name / school / DC / damage amount never appear on any player-visible surface. ✔
- Version-tolerant action read (`action.save ?? action.system?.save ?? action.data?.save`) avoids the
  PF1 11.11 `ItemAction#data` deprecation where direct access is available. ✔

## Notes / by-design
- This is the conservative slice of the 0.4.0 probe's "stripped-mechanics re-post" option. The probe
  flagged that surfacing full mechanics has real leak surface; this implementation deliberately
  surfaces only the three least-identifying facts, gated behind an informed opt-in, and excludes the
  save DC (which would reveal spell level) and damage amount/type (which would often reveal the spell).
- The effect line is presentation only: it is part of the already-public generic card content and is
  not stored in the registry or any secret store.
