# 0.5.0 Runtime Verification ÔÇö Spell Identification MVP

Status: **PASSED (GM-side, automation-driven, 2026-06-16)** for all privacy-critical and broker
logic; one player-seat hop remains for a live foreground confirmation (see Remaining). Driven via
in-page evaluation on the live GM client (user "Dade" = `game.users.activeGM`), Foundry `13.350` /
PF1 `11.11`. All touched state was restored afterward (chat back to baseline, spell slot restored,
public registry + private store cleared). Built on `docs/0.4.0-PROBE.md`.

## What was verified

### Suppression + privacy (the riskiest surface ÔÇö PASS)
- A non-player actor (NPC "Oswald", `hasPlayerOwner === false`) cast **Magic Missile**.
- **First attempt surfaced a real bug:** setting `shared.scriptData.hideChat` in `pf1PreActionUse`
  did NOT suppress the card ÔÇö PF1's `executeScriptCalls()` runs after that hook and rebuilds
  `scriptData`, clobbering the flag. The real Magic Missile card leaked alongside the generic one
  (`messagesDelta = 2`, one real `type:"action"` card containing "Magic Missile").
- **Fix:** suppress at **`pf1PreDisplayActionUse` returning `false`** (fires inside `postMessage`,
  after script calls, immediately before `ChatMessage.create`), tracking masked casts in a
  `WeakMap` keyed by the live ActionUse instance (PF1 cannot clobber it).
- **Re-test PASS:** `messagesDelta = 1` (only the generic card), `realSpellCardLeak = 0`. The single
  `preCreateChatMessage` event was the generic card (`hasMM:false`); no real spell card was created.
- The generic card carries **no** spell identity: content = "A spell is being cast.", `whisper:[]`,
  flags = `{ castId, kind:"maskedSpell" }` (a random castId only). `contentHasMM:false`,
  `flagsHasMM:false`.
- Public registry entry is safe-only: `{ state:"masked", identifiedBy:{}, publicLabel }` ÔÇö no name,
  school, uuid, or DC. The registry `spellcasting` JSON contains no secret.
- The true identity is stored ONLY in the client-scope private store (`privateStore` localStorage):
  `{ spellName:"Magic Missile", school:"evo", spellItemUuid, casterActorUuid, spellLevel:1, dc:16 }`.
  DC = **15 + spell level** (PF1 rule), derived at mask time.

### GM visibility (PASS)
- On the GM's client only, `renderChatMessageHTML` annotates the generic card (presentation-only,
  read from the private store ÔÇö never written to the document): note rendered as
  **"GM: Magic Missile ┬Ě Evocation ┬Ě Spellcraft DC 16"** plus a **"Request Spellcraft"** button.

### Broker + reveal (PASS, driven via the GM socket handler)
- Drove `executeAsGM("spellcraftResult", { castId, userId, actorId, actorName:"Pants", total:20 })`.
- `20 >= DC 16` Ôćĺ `markIdentified` recorded **safe** attribution
  (`identifiedBy[userId] = { actorId, actorName:"Pants", round, source:"spellcraft" }`) and the gate
  stayed `masked` (personal reveal; not globalized ÔÇö Decision #6).
- The reveal was a **Foundry-filtered whisper to the earner only**: content
  "You identified the spell: Magic Missile (Evocation).", `whisper:[<that user id>]`
  (`onlyToPlayer:true`). Non-identifying players never receive the document.
